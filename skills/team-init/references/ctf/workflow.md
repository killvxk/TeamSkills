# CTF 比赛团队工作流程

> CTF 比赛与常规开发截然不同：高度并行、时间敏感、以得分为驱动。
> 所有阶段围绕"最大化得分"展开，而非线性交付。

---

## 阶段总览

| 阶段 | 名称 | 时间占比 | 核心目标 | 参与角色 |
|------|------|----------|----------|----------|
| Phase 1 | 比赛准备 | 赛前 | 环境搭建、分工确认、工具就绪 | 队长主导，全员参与 |
| Phase 2 | 题目分配与并行攻坚 | 60% | 各方向选手同时解题，快速拿分 | 全员并行 |
| Phase 3 | 协作与交叉支援 | 25% | 卡题互助、跨方向协作、集中攻坚 | 按需组队 |
| Phase 4 | Flag 提交与验证 | 5% | 验证 flag 格式、提交得分、确认计分 | 队长 + 解题者 |
| Phase 5 | 复盘总结 | 赛后 | 撰写 writeup、总结经验、归档工具 | 全员 |

---

## Phase 1: 比赛准备

<phase id="1" name="比赛准备">

**目标：** 确保比赛开始时所有选手可以立即进入解题状态。

**流程：**
1. 队长建立比赛通信频道（IRC/Discord/群聊），确认所有人在线
2. 搭建共享环境：VPN 连接、共享文件夹、题目跟踪看板
3. 确认各选手方向分工（Web/Reverse/Crypto/PWN/Misc/Forensics）
4. 检查工具链就绪：IDA、Burp、pwntools、SageMath 等
5. 约定 flag 提交流程：解题者通知队长 → 队长统一提交
6. 制定时间节点：每 30 分钟状态同步，最后 1 小时集中攻坚

**完成条件：**
- [ ] 所有选手在线且通信正常
- [ ] 共享环境可用
- [ ] 工具链检查通过
- [ ] 分工与流程达成共识

</phase>

## Phase 2: 题目分配与并行攻坚

<phase id="2" name="题目分配与并行攻坚">

**目标：** 快速浏览所有题目，按方向分配，各选手同时开始解题，优先拿下低分易题。

**流程：**
1. 比赛开始后，队长快速浏览全部题目，按类别分类
2. 各方向选手领取对应题目，优先选择低分值题（签到题）
3. 每位选手独立攻坚，遇到进展或卡点及时在频道报告
4. 队长维护计分板：已解/进行中/未开始/放弃
5. 得分策略：先扫低分 → 再攻中等 → 高分题集中力量

**得分策略原则：**
- 签到题必须在前 30 分钟内全部拿下
- 中等题目由对应方向选手主攻，限时 1-2 小时
- 高分题目评估 ROI 后决定是否投入

**完成条件：**
- [ ] 所有题目已分配负责人
- [ ] 签到题全部完成
- [ ] 各方向至少有一道题在推进中

</phase>

## Phase 3: 协作与交叉支援

<phase id="3" name="协作与交叉支援">

**目标：** 对卡住的题目进行跨方向协作，集中优势力量攻克高分题。

**流程：**
1. 队长识别卡住超过 1 小时的题目，发起支援请求
2. 已完成本方向题目的选手转向支援
3. 跨方向题目（如 Web+Crypto 混合题）组建临时小组
4. 共享已发现的线索、半成品 exploit、部分 flag
5. 队长动态调整优先级：放弃 ROI 过低的题目

**完成条件：**
- [ ] 卡题均已获得支援或决定放弃
- [ ] 高分题有明确攻坚计划

</phase>

## Phase 4: Flag 提交与验证

<phase id="4" name="Flag 提交与验证">

**目标：** 确保 flag 正确提交，避免格式错误或重复提交导致罚分。

**流程：**
1. 解题者将 flag 发送给队长，附带简要解题过程
2. 队长核对 flag 格式（如 `flag{...}`、`CTF{...}`）
3. 提交至比赛平台，确认得分到账
4. 记录提交时间与得分，更新计分板

**完成条件：**
- [ ] 所有已解题目的 flag 均已成功提交
- [ ] 计分板与平台得分一致

</phase>

## Phase 5: 复盘总结

<phase id="5" name="复盘总结">

**目标：** 撰写 writeup，总结经验教训，为下次比赛积累知识库。

**流程：**
1. 每位选手为自己解出的题目撰写 writeup
2. 未解出的题目赛后复现，学习官方/他队 writeup
3. 队长汇总比赛数据：总分、排名、各方向得分占比
4. 团队讨论：哪些策略有效、哪些需要改进、工具链缺陷
5. 归档 writeup 和工具脚本到团队知识库

**完成条件：**
- [ ] 所有已解题目有 writeup
- [ ] 赛后复盘会议完成
- [ ] 经验教训记录归档

</phase>

---

## 关键原则

- **时间就是分数**：不要在单题上死磕，及时切换
- **并行最大化**：每位选手在任意时刻都应有明确任务
- **通信要快**：发现关键信息立即共享，不要等到整理完
- **先易后难**：签到题是基本盘，高分题是锦上添花
- **动态调整**：根据剩余时间和已得分数实时调整策略
